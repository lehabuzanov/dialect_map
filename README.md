# Интерактивная карта русских говоров Удмуртии

Проект запускается напрямую через Streamlit. Карта формируется во время запуска приложения, без отдельной статической сборки.

## Что используется

- `streamlit_app.py` — точка входа приложения.
- `requirements.txt` — зависимости для Streamlit Cloud.
- `scripts/data_loader.py` — загрузка и нормализация рабочей таблицы. Приоритетный локальный файл: `data/csv/dialect_map_data_answers_v2.csv`, резервный: `data/csv/dialect_map_data.csv`.
- `scripts/area_generator.py` — построение предварительных ареалов и изоглосс.
- `scripts/page_renderer.py` — подготовка HTML-страницы карты.
- `web/` — шаблон, стили и JavaScript интерфейса.
- `notes/05_ui_notes.txt` — инструкция пользователя.

## Основная таблица данных

Приложение работает от одной итоговой таблицы:

- `data/csv/dialect_map_data_answers_v2.csv`
- резервный старый формат: `data/csv/dialect_map_data.csv`

Структура строк:

- `region` — регион.
- `district` — район или городской округ.
- `settlement` — населённый пункт.
- `lat` — широта.
- `lon` — долгота.
- `question` — полный текст вопроса с пометкой атласа (`ЛАРНГ:` или `ДАРЯ:`).
- `unit1` — основной вариант ответа.
- `unit2` — дополнительный вариант ответа.
- `comment` — краткое примечание.

## Логика приложения

- Вопросы автоматически делятся на два атласа: `ЛАРНГ` и `ДАРЯ`.
- На карте всегда показываются реальные координаты населённых пунктов.
- При выборе одного вопроса показываются связанные точки и ареал этого вопроса.
- При выборе двух вопросов каждому вопросу назначается свой цвет, а между ними показываются изоглоссы.
- Карточки населённого пункта, района и вопроса собираются из данных этой же таблицы.

## Запуск в Streamlit

```bash
streamlit run streamlit_app.py
```

Для публикации в Streamlit Community Cloud достаточно загрузить репозиторий в GitHub и указать `streamlit_app.py` как entrypoint.
